  • 模块化(如何在不同模块下实现信息调用?)
    • 变量的信息
      • 连接性
      • 作用域
        • 定义位置
          • 函数内部定义(函数内部才能使用)
          • 函数内部复合语句中(块作用域)
          • 全局变量(文件作用域)
        • 例: f1(int a,int b)只能在f1内部使用
          • main函数和其他函数一样
          • 形参也是在函数内部定义的
          • 不同函数间的变量名可以相同,因为函数活动记录不同
        • 复合语句中的变量可以和函数中的变量名一样,以下是合法的,函数的形参被在复合作用域中隐藏
          • f(int x)
          • while(1) int x=1;x++; printf(%d",x); if(x==3) break;}
        • 全局变量不要用
          • 原因1:很难知道全局变量在哪个函数被修改导致错误
          • 原因2:导出函数需要包括全局变量
      • 存储类别和存储期
        • auto
          • 只能修饰局部变量,默认的
          • 内存分配:栈区
          • 不初始化 值为垃圾值
          • 出了作用域立刻销毁
        • regiser
          • 变量放在寄存器中,也就是CPU里面,快
          • 不能被其他文件访问
          • 现代不一定分配到寄存器中,主要看编译器心情
        • static
          • 不会被释放 (直到程序释放)
          • int f1() static int n; n=n*2;连续调用f1 5次后,不会重复初始化
          • 静态全局变量,不能被其他文件访问
        • extern
          • 一般全局变量可以被外部文件访问
          • extern int out;//声明
          • 声明和定义不同,声明时无需分配内存空间,extern只是声明我调用了全局变量
        • 总结
          • 动态声明:auto||register
          • 静态声明:全局变量实现
        • 函数的声明
          • 静态关键字:static
          • 作用:不用担心重名
          • 如何声明函数在外部可以使用?
            • 1.extern关键词(默认)
            • 2.头文件引入
  • 模块化
    • 步骤:
      • 1.自顶向下,逐步细化
      • 2.细化的问题用模块构建
      • 3.组成一个函数解决问题
    • 概要设计方式:
      • 没有绝对的划分方法
        • 高内聚 低耦合
          • 内聚:单个模块的功能强度
          • 耦合:模块间互相连接的强度
      • 高内聚:
        • 1.函数实现简单
        • 2.函数接口简单
        • 3.函数返回值少(只返回一个值)
      • 低耦合:
        • 1.不使用全局变量
        • 2.函数参数尽可能少
        • 3.尽量少使用复杂数据类型
      • 通用方法
        • 1.输入处理
        • 2.输出处理
        • 3.数据处理
        • 4.文件总调度(主函数完成)

        • 把所有需要访问的全局变量,标准头文件,函数放在一个自定义的头文件中
        • typedef
        • #define

      • 防止头文件被重复引用
